Vintage Logistics Website & Mobile App

June 2018 - July 2018

Vintage Logistics is a third-party logistics company based in Laredo, Texas. Vintage uses software from IMEXSOL (an import/export software solutions company) to manage information about their customers' shipments. This project consisted of two key tasks.

Firstly, I redesigned Vintage's website to be more modern, device-responsive, and generally more visually appealing. Secondly, an important problem that they needed to solve was their lack of an online shipment tracking system for their customers. Vintage's customers had to call or e-mail to request the status on a certain shipment. I solved this problem by bridging the website with Vintage's database through a newly constructed API that I access from the site's front-end.

Customers can now log on to their website and sign in with credentials provided by Vintage in order to track their shipments.

As part of my efforts to help modernize and improve the company's software-based resources, I am developing an Android and iOS application for Vintage Logistics.

Evacuation Shelter and Response Decision Tool

Jan 2018 - May 2018

The American Red Cross has developed a standard 120-hour timeline of critical decision points in their response to the event of a hurricane. The limit to this timeline is that it is not particularly user-friendly - it is stored in an excel file and is not adjustable based on variables known only for that particular disaster. This application seeks to enable American Red Cross managers to view the response timeline in a user-friendly fashion in order to response to the disaster more quickly and efficiently. The app would also allow users to adjust the time it takes for certain steps in the response and it would also send the user notifications regarding the time left to complete certain tasks. This project is be designed to facilitate disaster relief efforts by the American Red Cross.

Polygon Rasterizer

Sep 2017 - Sep 2017

This is a polygon rasterizer than I wrote for Computer Graphics. The input is a .obj file, which is a text file format containing x, y, and z values for vertices with coordinate values between -1.0 and 1.0 (along with other info), and the output is the rendered 3D model. The program performs all the steps in the rendering process: lighting calculations, coordinate system transformations, scan conversions, and color interpolation across polygons for smooth shading.

Ray Tracer

Nov 2016 - Nov 2016

This is a ray tracer that I wrote for Computer Graphics as well. It draws surfaces given their implicit equations, so they are as smooth as possible no matter how much you might zoom in. The ray tracer works by running a for loop through every pixel, taking a parametric line and intersecting it with all the surfaces in the scene, then filling in the appropriate color based on the shape's properties. The surfaces may be infinite cylinders, spheres, ellipsoids, or infinite planes. The ray tracer also functions recursively in order to calculate reflections between surfaces, as shown on the bottom right image.

Repository is located here.
